Abstract Data Types (ADTs)

An ADT is a tuple （T, U, O, P, A） where

  • T is the name of the ADT we are defining.

  • U is a set of （already defined） ADTs used in the definition of T. Notice that this implies a notion of a hierarchy of ADTs.

  • O is a set of operation signatures that defines the syntax for the legal use of T objects.

  • P is a set of pre-conditions （logical expressions that always must hold true） to using some of the operations in O.

  • A is a set of axioms （logical expressions that are admittedly true） defining the behaviour of the operations in O.

Operation signature

The signature of an operation is made of

  1. its name;
  2. a （possibly empty） set of ADT describing the domains for parameter values for the operation;
  3. an ADT describing the domain of the result of the operation. The ADT specifying the parameter types are called input ADT （or source ADT）, and that specifying the result is called output （or target） ADT.

For example:

ADT boolean

operations

true: -> boolean

false: -> boolean

$!_$: boolean -> boolean

$_∧_$: boolean x boolean -> boolean

$_∨_$: boolean x boolean -> boolean

axioms

$!true = false$

$!!a = a$

$a ∧ true=a$

$a ∧ false = false$

$a ∨b=!(!a∧ !b)$

Axioms:

$A_1: 0+x \equiv x$

$A_2: sx+y \equiv x + sy$

$A_3: 0=0 \equiv true$

$A_4: 0=sx \equiv false$

$A_5: sx=0 \equiv false$

$A_6: sx = sy \equiv x=y $

Example: Is 2 = 3? Proof with axiom
$$
ss0 &= sss0…A_6\\
s0 &= ss0…A_6\\
0 &= s0 \rightarrow false …A_3
$$

ADT counter

a counter has a natural value (0 at inial time). It can only be incremented or decremented by 1 each time. Decrementing a null has no effect

Operations:

new: -> counter

increment: counter-> counter

decrement: counter-> counter

value: counter -> natural
$$
&A_1: dec(new) \equiv new\\
&A_2: dec(inc(c)) \equiv c…\text{both c and new are counters}\\
&A_3: value(new) \equiv 0\\
&A_4: value(inc(c)) \equiv value(c)+1
$$
another example: $value(inc(dec(inc(new))))=value(0+1-1+1)=0$

ADT Stack

A stack in computer science can be seen as a homogeneous collection of elements for which only the topmost element is accessible. For instance, a stack of plates is almost always accessed by its top, be it to take plates before lunch or dinner, or to put them back after washing up. One commonly uses the term LIFO （for Last In, First Out） to refer to that kind of data structure:

• Tell wether the stack is empty.

• Push a new element on top of the stack.

• Pop the top element out of the stack.

• Tell what the top element is.

ADT Stack Uses Element, boolean

Operations

$create : -> Stack$

$isEmpty: Stack -> boolean$

$push:Stack x Element - Stack$

$pop: Stack Stack$

$top:Stack -> Element$

Preconditions

$pop（p） \iff !isEmpty（p)$

$top（p） \iff !isEmpty（p)$

Axioms

$A_1: isEmpty (create) = true$

$A_2: isEmpty (push(p,e))= false$

$A_3: top (push(p,e))= e$

$A_4: pop(push(p,e)) = p$
