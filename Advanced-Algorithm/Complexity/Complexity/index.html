Intro to Complexity

In CS, we have different ways to solve the same question. Depending on the chosen criteria, one of them is the “best” solution. An algorithm is defined as complex based on the amount of Space and Time it consumes. Hence the Complexity of an algorithm refers to the measure of the time it will need to execute and get the expected output and the Space it will need to store all the data (input, temporary data, and output). Hence these two factors define the efficiency of an algorithm.

  • Time Factor: Time is measured by counting the number of key operations such as comparisons in the sorting algorithm.
  • Space Factor: Space is measured by counting the maximum memory space the algorithm requires to run/execute.

Analysis of Complexity

Asymptotic notation is a way to describe the running time or space complexity of an algorithm. The three most commonly used notations are Big O, Big Omega, and Big Theta.

Big O Notation: Big $O$ notation represents the upper bound of the running time of an algorithm. Therefore, it gives the worst-case complexity of an algorithm.

Big $\Omega$ Notation: Big $\Omega$ notation represents the lower bound of the running time of an algorithm. Thus, it provides the best-case complexity of an algorithm.

Big $\Theta$ Notation: Big $\Theta$ represents the upper and the lower bound of the running time of an algorithm, it is used for analyzing the average-case complexity of an algorithm.

In real life, we will consider the worst situation when analyzing an algorithm because it is harder and impossible (sometimes) to consider the other two.

Mathematic Algorithm

We can represent the asymptotic notation in a mathematical way with the help of the Limit.

$f(n) \in O (g(n)) \Rightarrow \displaystyle \lim_{n \to \infty} f(n)/g(n) \in [0,\infty) $, in other words, f(n) is the worst case of g(n)

$f(n) \in \Omega(g(n)) \Rightarrow \displaystyle \lim_{n \to \infty} f(n)/g(n) \in (0,\infty] $

$f(n) \in \Theta(g(n)) \Rightarrow \displaystyle \lim_{n \to \infty} f(n)/g(n) \in (0,\infty) $

Properties

General Properties: If $f(n)$ is $O(g(n))$ then $a \cdot f(n)$ is also $O(g(n))$ where $a$ is a constant.

Transitive Properties: if $f(n) \in O(g(n)), g(n) \in O(h(n))$, then $f(n) \in O(h(n))$.

Reflexive Properties: $f(n) \in O(f(n))$, also $f(n) \in \Theta(f(n))$

Symmetric Properties: if $f(n) \in \Theta(g(n))$, then $g(n) \in \Theta(f(n)$

Transpose Symmetric Properties: if $f(n) \in O(g(n))$, then $g(n) \in \Theta(f(n))$

Calculation of Time Complexity

We usually focus on time complexity.

$O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)$

Constant Time Complexity $O(1)$:

The time complexity of a function (or set of statements) is considered as $O(1)$ if it doesn’t contain a loop, recursion, and call to any other non-constant time function.

Linear Time Complexity $O(n)$:

The Time Complexity of a loop is considered as $O(n)$ if the loop variables are incremented/decremented by a constant amount.

Quadratic Time Complexity $O(n^n)$:

The time complexity is defined as an algorithm whose performance is directly proportional to the squared size of the input data, as in nested loops it is equal to the number of times the innermost statement is executed.

Logarithmic Time Complexity $O(\log n)$:

The time Complexity of a loop is considered as $O(\log n)$ if the loop variables are divided/multiplied by a constant amount. Also for recursive calls in the recursive function, the Time Complexity is considered as $O(\log n)$.

Calculation of complexity

The complexity of a sequence is the sum of each statement included in the sequence

For the loop, the complexity $C = \displaystyle\sum_i C_i$, $i$ is the index of the statement inside of the loop.

For the condition, the complexity:

  • worst case: $C_i=C_{cond} + max(C_{if}+C_{else})$

  • best case: $C_i=C_{cond} + min(C_{if}+C_{else})$

  • average case: $C_i=C_{cond} + P*C_{if}+(1-P)*C_{else}$

Complexity of recursion

To determine a recurrence equation, we need to find a relationship between the initial call and the closer call to the stop.

Example:

Consider this recursion code:

$$
\begin{cases}
C_{0} = C_{1} = 0\\
C_{n} = C_{n-1} + 1 = C_{n-2}+1+1 = … =n-1
\end{cases}
$$

Divide and conquer

$ C_n = a C_{\frac n b}+\theta (n^c)$

Master theorem:

  • $a<b^c: C_n = \theta (n^c)$
  • $a=b^c: C_n = \theta (n^c\log n)$
  • $a>b^c: C_n = \theta (n^{\log_b^a})$

For ipow: $C_n = C_{\frac n 2}+1 = \theta (\log n)$

For merge sort: $C_n = 2(\frac n 2)+n = \theta(n\log n)$

For Fibonacci sequence:

A Fibonacci sequence has the formation as follows:
$$
\begin{cases}
f_0 = f_1 = 1\\
f_n = f_{n-1} +f_{n-2}
\end{cases}
$$

$$
\begin{cases}
C_0 = 0\\
C_1 = 0\\
C_n =1+ C_{n-1} +C_{n-2}
\end{cases}
\Rightarrow\\
let\quad D_n = C_n+1 \quad(\text{substitute })\Rightarrow\\D_n = D_{n-1}+D_{n-2}\\
$$

Amazing, it is still a Fibonacci sequence! Suppose $D_n = \alpha^n$ , then we can get $\alpha^n = \alpha^{n-1}+\alpha^{n-2}$, suppose $n=2$, then $\alpha = \frac {1\pm \sqrt{5}} 2$, that means $D_n = ({\frac {1\pm \sqrt{5}} 2})^n$.

To avoid recomputation, let’s reform the code like this:

$$
\begin{cases}
C_0 &= C_1 =0\\
C_n &= C_{n-1}+1\\
&=C_{n-2}+2\\
&=C_{n-3}+3\\
&=\cdot\cdot\cdot \\
&=C_{n-k} + k\\
&=\theta(C_n)
\end{cases}
$$
