Abstract Data Types (ADTs)

An ADT is a tuple （T, U, O, P, A） where

  • T is the name of the ADT we are defining.

  • U is a set of （already defined） ADTs used in the definition of T. Notice that this implies a notion of a hierarchy of ADTs.

  • O is a set of operation signatures that defines the syntax for the legal use of T objects.

  • P is a set of pre-conditions （logical expressions that always must hold true） to using some of the operations in O.

  • A is a set of axioms （logical expressions that are admittedly true） defining the behaviour of the operations in O.

Operation signature

The signature of an operation is made of

  1. its name;
  2. a （possibly empty） set of ADT describing the domains for parameter values for the operation;
  3. an ADT describing the domain of the result of the operation. The ADT specifying the parameter types are called input ADT （or source ADT）, and that specifying the result is called output （or target） ADT.

For example:

ADT boolean

operations

true: -> boolean

false: -> boolean

!_ : boolean -> boolean

_∧_ : boolean x boolean -> boolean

∨: boolean x boolean -> boolean

axioms

$!true = false$

$!!a = a$

$a ∧ true=a$

$a ∧ false = false$

$a ∨b=!(!a∧ !b)$

Axioms:

$A_1: 0+x \equiv x$

$A_2: sx+y \equiv x + sy$

$A_3: 0=0 \equiv true$

$A_4: 0=sx \equiv false$

$A_5: sx=0 \equiv false$

$A_6: sx = sy \equiv x=y $

Example: Is 2 = 3? Proof with axiom
$$
\begin{align*}
ss0 &= sss0…A_6\\
s0 &= ss0…A_6\\
0 &= s0 \rightarrow false …A_3
\end{align*}
$$

ADT counter

a counter has a natural value (0 at inial time). It can only be incremented or decremented by 1 each time. Decrementing a null has no effect

Operations:

new: -> counter

increment: counter-> counter

decrement: counter-> counter

value: counter -> natural
$$
\begin{align*}
&A_1: dec(new) \equiv new\\
&A_2: dec(inc(c)) \equiv c…\text{both c and new are counters}\\
&A_3: value(new) \equiv 0\\
&A_4: value(inc(c)) \equiv value(c)+1
\end{align*}
$$
another example: $value(inc(dec(inc(new))))=value(0+1-1+1)=1$

ADT Stack

A stack in computer science can be seen as a homogeneous collection of elements for which only the topmost element is accessible. For instance, a stack of plates is almost always accessed by its top, be it to take plates before lunch or dinner, or to put them back after washing up. One commonly uses the term LIFO （for Last In, First Out） to refer to that kind of data structure:

• Tell wether the stack is empty.

• Push a new element on top of the stack.

• Pop the top element out of the stack.

• Tell what the top element is.

ADT Stack Uses Element, boolean

Operations

$create : -> Stack$

$isEmpty: Stack -> boolean$

$push:Stack\quad \text{x}\quad Element - Stack$

$pop: Stack Stack$

$top:Stack -> Element$

Preconditions

$pop（p） \iff !isEmpty（p)$

$top（p） \iff !isEmpty（p)$

Axioms

$A_1: isEmpty (create) = true$

$A_2: isEmpty (push(p,e))= false$

$A_3: top (push(p,e))= e$

$A_4: pop(push(p,e)) = p$

ADT Queue

A queue is an homogeneous collection of elements accessed by one of the two ends of the structure: insertion at the one end, removals at the other end.

Intuitively, it mirrors the behaviour of a real queue, such as the one that can appear at the entrance of a cinema for instance. It can be called FIFO, for First In, First Out: the first element that entered the queue is the first to leave it.

Expected functionalities are as follows:

• Tell wether the queue is empty

• Add an element to the end of the queue

• Remove the first （head） element from the queue

• Tell which is the head element

Uses boolean,Element

operations

$create: queue$

$isEmpty:queue \to boolean$

$add : queue \quad \text{x}\quad Element \to queue$

$remove:queue \to queue$

$head:queue \to Element$

pre-conditions

$remove (f, e)\iff !isEmpty(f)$

$head (f)\iff !isEmpty(f)$

axioms

$A_1:isEmpty （create） \equiv true$

$A_2: isEmpty(add(f,e)) \equiv false$

$A_3: isEmpty(f) \Rightarrow head(add(f,e))\equiv e$

$A_4: !isEmpty （f） \Rightarrow head(add(f, e))= head(f)$

$A_5: isEmpty(f) \Rightarrow remove(add(f,e))= create$

$A_6:!isEmpty(f)\Rightarrow remove(add (f, e))= add(remove (f),e)$

Tower of Hanoi

hanoi(n,from,to,use) n is the amount of disks, from is the inital stick, to is the destination stick, use is the aux stick

List ADT

Inside a list, each element is associated with its （unique） position （a strictly positive integer）. Unlike the previous two ADT, any element of the list can be accessed, whatever its position is.

The minimum functionalities expected from a basic list are as follows:

• access the top element （first） of the list

• access the rest of the list （i.e.the list minus its first element） • tell the length of the list

• access an element based on its position in the list

• add an element at a given position in the list

• remove an element at a given position in the list

As a first step, we just define the ADT with its constructors and access functions （accessors） to the frst element and to the rest of the list （the other operations can then be expressed using those basic ones):

ADT List

Uses boolean,Element,Natural operations

$create : \to list$

$isEmpty: list \to boolean$

$rest : list \to list$

$cons : Element \quad\text{x}\quad list \to list$

$first: list \to Element$

pre-conditions

$first(l)\iff !isEmpty(l)$

$rest(l)\iff !isEmpty(l)$

axioms

$A_1: isEmpty（create） \equiv true$

$A_2: isEmpty (cons(e, l))\equiv false$

$A_3: first(cons(e, l))\equiv e$

$A_4:rest(cons(e, l)) \equiv l$

Minimum list extension

Based on the functions described above, we can now define all the classical operations about lists. We get the following ADT:

extension ADT list

Uses Natural

operations

$length:1ist \to Natural$

$add : list \quad\text{x}\quad Natural \quad\text{x}\quad Element \to list$

$remove :list \quad{x} Natural \to list nth : list x Natural Element$

pre-conditions

$remove (1,n)) \iff 1 ≤n ≤ length(1)$

$n^{th}(l,n))\iff 1≤n ≤ length(1)$

$add（1,n,e））\iff 1 ≤n ≤ length（1） + 1$

axioms

$A_5: length（create） = 0$

$A_6: 1!=create \Rightarrow length(1) \equiv length(rest(1))+ 1$

$A_7: n^{th}（1,1） \equiv first(1)$

$A_8: i>1=n^{th}(1,i) \equiv n^{th}(rest(1),i-1)$

$A_9: add(1,1,e) \equiv cons(e,1)$

$A_{10}:i>1=add（l,i,e）\equiv cons (first(1),add(rest(1),i-1,e)$

$A_{11}: remove (1,1) = rest(1)$

$A_{12}: i>1 \Rightarrow remove(1,i) \equiv cons(first(1),remove (rest(1),i-1)$

Merging sorted list

ADT merged list

Use element in lists

Operation

$issorted: list \to boolean$

$merge: list \quad\text{x}\quad list \to list$

Precondition

$merge(l_1,l_2) \iff issorted(l_1) \quad\text{and}\quad issorted(l_2)$

Axioms

$A_1: issorted(new)\equiv T$

$A_2: issorted(cons(l,new)) \equiv T$

$A_3: issorted(cons(e,l))\equiv e<=first(l) \quad\text{and}\quad issorted(l)$

$A_4:issorted(l) \equiv first(l) <=first(rest(l)) \quad\text{and}\quad issorted(rest(l))$

$A_5: merge(new,l) \equiv l$

$A_6: merge(l,new)\equiv l$

$A_7: first(l_1)<=first(l_2) \Rightarrow merge(l_1,l_2) \equiv cons(first(l_1),merge(rest(l_1),l_2))$

$A_8: first(l_1)>first(l_2) \Rightarrow merge(l_1,l_2) \equiv cons(first(l_2),merge(rest(l_2),l_1))$

ADT indexof list

operations

$indexof: list \quad \text{x}\quad element \to Natutral$

Precondition

$indexof(l,e) \iff cons(l,e)$

Axioms

$A_1: indexof(new,e)\equiv 0$

$A_2: indexof(cons(e,l),e) \equiv 1$

$A_2^*:indexof(l,first(l))\equiv 1$

$A_3: indexof(cons(e_1,l,e_2))\equiv l+indexof(l,e_2)$

$A_3^*:indexof(l,e)\equiv 1+indexof(rest(l),e)$

ADT index3(3 refers to 3 parameters in total and this function is used to check the postion of the 3^th element)

Operations

$index3: list \quad\text{x}\quad element\quad\text{x}\quad natural \to natural$

Axiom

$A_1: indexof(l,e)\equiv index3(l,e,1)$

$A_2: index3(new,e,n)\equiv 0$

$A_3: index3(cons(e,l),e,n) \equiv n$

$A_4:index3(cons(e_1,l),e_2,n)\equiv index3(l,e_2,n+1)$

ADT revere list

$last((A,B,C,D)) \to D$

$allbutlast((A,B,C,D))\to(A,B,C)$

Operations

$last: list \to element$

$allbutlast: list \to list$

$reverse: list \to list$

Axioms

$A_1: last(cons(e,new))\equiv e$

$A_2: last(cons(e,l))\equiv last(l)$

$A_3: allbutlast(cons(e,new))\equiv new$

$A_4: allbutlast(cons(e,l)) \equiv cons(e,allbutlast(l))$

$A_5: reverse(new) \equiv new$

$A_6: reverse(l) \equiv cons(last(l),reverse(allbutlast(l)))$

Complexity

last:

$C(1) = 1$

$C(n) =1+C(n-1)=2+C(n-2)+…+i+C(n-i)$, suppose $i =n-1$, then we get $C(n)=n$

allbutlast:

$C(1) = 0$

$C(n)=C(n-1)+3 =C(n-2)+6 =…=C(n-i)+3(n-i)$,suppose $i = n-1$, then we get $C(n) = 3(n-1)$

reverse:

$C(0) = 0$
$$
\begin{align*}
C(n) &= 1+n+3(n-1)+C(n-1)\\
&=C(n-1)+4n-2\\

&=C(n-2)+4(n+(n-1))-2-2\\
&=C(n-3)+4(n+(n-1)+(n-2))-2-2-2\\
&=…\\
&=C(n-i)+4\sum_{k=n-i+1}^nk-2i\\
&=C(0)+4\frac {n(n+1)} 2 -2n ….\text{(suppose i =n)}\\
&=2n^2

\end{align*}
$$

Trees

A tree is composed of a node （called its root） containing information, and of a（possibly empty） set of trees （called its children）. This definition is clearly recursive, as will be most of the definitions of operations on trees.

ADT Tree

Use integer

Oporations

$new: element \times forest \to tree$

$newforest: forest$

$addtree: forest \times tree\times integer \to forest$

$root: tree \to element$

$subTrees: tree \to fortest$

$ntree: forest \to integer$

$nth: forest \times interger \to tree$

$remTree: forest \times integer \to forest$

** Preconditions**

$nth(f,n) \iff 1\le n \le nTree$

$newTree(f,n)\iff 1\le n \le nTree$

$addTree(f,t,n) \iff 1\le n \le nTree+1$

Axioms

$A_1: root(new(e,f))\equiv e$

$A_2: subTrees(new(e,f))\equiv f$

$A_3: nTrees(newF) \equiv 0$

$A_4: nTrees(addTree(f,t,n)) \equiv nTrees(f)+1$

$A_5: a>n \Rightarrow nth(addTree(f,t,a),n) \equiv nth(f,n)$

$A_6: nth(addTree(f,t,n),n) \equiv t$

$A_7: nth(addTree(f,t,a),n) \equiv nth(f,n-1)$

Explanation: the left part says to add a new subtree to the forest, and n refers to the rank after modification. So, if to find the same subtree of the forest before modification, we need to minus 1 to eliminate the influence of rank from adding a new subtree

$A_8:a>r \Rightarrow remTree(addTree(f,t,a),r)\equiv addTree(remTree(f,r),t,a-1)$

$A_9:remTree(addTree(f,t,n),n)\equiv f$

$A_{10}: remTree(addTree(f,t,a),r)\equiv addTree(remTree(f,r-1),t,a)$

ADT extension tree, forest

Use Boolean

Operations

$contains: tree \times element \to boolean$

$containsF: forest \times element \to boolean$

Axioms

$A_1: contains(new(e,f),e) \equiv true$

$A_2:contains(t,e) \equiv containsF(subTrees(t),e)$

$A_3: containsF(newF,e)\equiv false$

$A_4:containsF(addTree(f,t,n),e) \equiv contains(t,e) \or containsF(f,e)$

Excersise:

For ADT Tree

size(t) $\to$ numbers of t

height(t) $\to$ height of t

PL(t) $\to$ pathlength of t

For ADT extension Tree, Forest

Opertaion

$size \to Int$

$sizeF \to Int$

$height: int$

$heightF: \to int$

$PL: \to int$

$PLF: \to int$

Axioms

$A_1: size(new(e,f)) \equiv 1 + sizeF(f)$

$A_2: sizeF(newF) \equiv 0$

$A_3: sizeF(addTree(f,t,n))\equiv size(t)+sizeF(f)$

$A_4: height(new(e,f)) \equiv 1+height(f)$

$A_5: heightF(newF) \equiv 0$

$A_6: heightF(addTree(f,t,n)) \equiv max(height(t),heightF(f))$

$A_7: PL(new(e,f))\equiv PLF(f)+sizeF(f)$

$A_8: PLF(newF) \equiv 0$

$A_9: PLF(addTree(f,t,n))\equiv PLF(f)+PL(t)$

Binary Tree

Binary trees are one of the simplest forms of tree structures. They are of great importance because they offer a rather simple way to obtain a good level of performance （in average） when searching, adding or removing information.

Change tree to binary tree:

$root \to root$

$1st \text{ subtree } \to left child$

$right \text{ sibling } \to right child$

Important properties on binary trees

  1. For a binary tree of size $n$ and height $h$, we have

$\log_2n \le h \le n-1$

  1. For a binary tree of size $n$ and path length $PL$, we have

$\sum_{i=1}^n|\log_2i|\le PL\le \frac {n(n-1)} 2$

  1. Average height of a BT of size n

$h(t) = \frac 1 n \sum_{e \in t}(h(e))= \frac {PL(t)} {size(t)}$

ADT Binary Tree(BT)

Use boolean, element

Opreations

$new: \to BT$

$element \times BT \times BT \to BT$

$BT \to element$

$left: BT \to BT$

$right: BT\to BT$

$contains: BT \times element \to boolean$

Preconditions

$root(t)\iff t \neq new$

$left(t)\iff t \neq new$

$right(t)\iff t \neq new$

Axioms

$A_1: root(<r,L,R>) \equiv r$

$A_2: left(<r,L,R>) \equiv L$

$A_3: righ(<r,L,R>) \equiv R$

$A_4: contains(new,e)\equiv false$

$A_5: contains(<r,L,R>,r) \equiv true$

$A_6: contains(<r,L,R>,e)\equiv contains(L,e) \or contains(R,e)$

ADT extension BT

Use interger

Operations

$size: BT \to integer$

$height: BT \to integer$

$PL: BT \to integer$

Axioms

$A_1:size(new) \equiv 0$

$A_2: size(<r,L,R>) \equiv 1+size(L)+size(R)$

$A_3: height(new) \equiv -1$

$A_4: height(<r,L,R>) \equiv 1+max(height(L),height(R))$

$A_5: PL(new) \equiv 0$

$A_6: PL(<r,L,R>)\equiv PL(L)+size(L)+PL(R)+size(R)$

BST Search Tree

In a BST, elements are stored in such a way that, for any node n,

  • all elements in the left child have a key value less than that of n,
  • all elements in the right child have a key value greater than that of n.

ADT BST extend BT

Uses Element,boolean

operations

$contains: BST \times Element \to boolean$

axioms

$A_1:contains（create,e） \equiv false$

$A_2:contains（<r,L,R>,r）\equiv true$

$A_3:key（x）<key（r）\Rightarrow contains（<r,L,R>,x）= contains (L,x)$

$A_4: key（x）>key（r）\Rightarrow contains（<r,L, R>,x）= contains(R, x)$

Leaf addition in a BST

ADT extension BST

Operations

$add: BST \times e \to BST$

Axioms

$A_1: add(create,e) \equiv <e,create,create>$

$A_2: add(<e,L,R>,e)\equiv <e,L,R>$

$A_3: add(t,root(t))\equiv t$

$A_4: r<e \Rightarrow add(<r,L,R>,e) \equiv (r,L,add(R,e))$

$A_5: r>e \Rightarrow add(<r,L,R>,t)\equiv (r,add(L,e),R)$

Complexity

$C(0)=0$

$C(h) = C(h-1)+3=C(h-2)+6=…=C(h-i)+3i = 3h$

Idea: use a BST to sort a set of numbers.

  1. build the BST from this set

$C(n) = \sum_{i=1}^n(add(i))=\Theta(\sum_{i=1}^n logi) = O(nlog(n))$

  1. inorder the BST to set the sorted list

$C(sort(n)) = O(nlogn)$

Extensions to ADT BST with the following operations

$lessthan(t,e) \to$ a BST with all elements in $t$ that are less than $e$

$greaterthan(t,e) \to$ a BST with all elements in $t$ that are greater than $e$

$rootAdd(t,e) \to$ a BST with $e$ as the root and all elements of $t$

Operations:

$lessthan: BST \times Element \to BST$

$greaterthan: BST \times Element \to BST$

$rootAdd:BST \times Element \to BST$

Axioms:

$A_1: lessthan(new,e)\equiv new$

$A_2: lessthan(<r,L,R>,r) \equiv L$

$A_3: e<r,lessthan(<r,L,R>,e) \equiv lessthan(L,e)$

$A_4: e>r, lessthan(<r,L,R>,e)\equiv <r,L,lessthan(R,e)>$

$A_5: greaterthan(new,e)\equiv new$

$A_6: greaterthan(<r,L,R>,r)\equiv R$

$A_7: r<e,greaterthan(<r,L,R>,e)\equiv greaterthan(R,e)$

$A_8:e<r,greaterthan(<r,L,R),e) \equiv <r,greaterthan(L,e),R>$

Extension to ADT BST with the 3 operations

$max(t)$: return the max element of $t$

$delmax(t)$: return $t$ minus its max element

$remove(t,e)$: return $t$ minus $e$

ADT extension BST

Operations:

$max(t): BST \to element$

$delmax(t): BST \to BST$

$remove(t, e): BST \times element \to BST$

Preconditions

$max(t) \iff !\equiv new$

Axioms

$A_1: max (<r,L,create>)= r$

$A_2: R\neq create \Rightarrow max(<r,L,R>)\equiv max(R)$

$A_3: delMax(create) \equiv create$

$A_4: delMax (<r,L,create>)\equiv L$

$A_5: L \neq create \Rightarrow delMax （<r,L, R>）\equiv <r, L, delMax（R）>$

$A_6: remove（create,e）\equiv create$

$A_7: key （x）<key（r）\Rightarrow remove （<r,L, R>,x）\Rightarrow <r,remove（L, x）,R>$

$A_8: key （x）>key(r) \Rightarrow remove （<r,L, R>, x）\equiv <r,L,remove(R, x)$

$A_9: remove （<r,create,R>,r）\equiv R$

$A_{10}: L \neq create \Rightarrow remove （<r, L, R>,r）\equiv <max （L）,delMax （L）,R>$

ADT AVL extend BST

Operations

$rRot: BST \to AVL$

Precondition

$rRot(t) \iff t ! \equiv new \text{and} left(t) !\equiv new \text{and} left(left(t)) !\equiv new$

Axioms

$rRot(<r,<lr,<llr,lll,llr>,lr>,r>) \equiv <lr,<llr,lll,llr>,<r,lr,r>>$
